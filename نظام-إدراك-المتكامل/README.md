# مشروع إدراك (Idrak) - الحالة الراهنة للمشروع

## نظرة عامة
تطبيق "إدراك" هو منصة تعليمية ذكية تعتمد على تحويل الصوت إلى نصوص ومعالجتها باستخدام الذكاء الاصطناعي.

## البنية التقنية للحفظ (Hybrid Saving)
يعتمد التطبيق نظام حفظ مزدوج يفرق بين نوعين من المستخدمين:
1. **المستخدم الضيف (Guest):**
   - يتم التعرف عليه عبر معرف يبدأ بـ `guest-`.
   - يتم تخزين بياناته محلياً في المتصفح باستخدام `localStorage` (المفتاح: `idrak_guest_items`).
2. **المستخدم المسجل (Authenticated):**
   - يتم تخزين بياناته سحابياً في قاعدة بيانات **Supabase**.
   - يتم التحقق من هويته عبر نظام `supabase.auth`.

## قاعدة البيانات (Supabase)
- **الجدول الرئيسي:** `items`
- **الأعمدة:**
  - `id`: معرف فريد (Primary Key).
  - `title`: عنوان الجلسة.
  - `content`: المحتوى النصي (Transcript).
  - `user_id`: معرف المستخدم (Foreign Key).
  - `created_at`: تاريخ الإنشاء.

## الأمان والخصوصية
- تم تفعيل سياسات **RLS (Row Level Security)** على جدول `items`.
- السياسات تضمن أن كل مستخدم (مسجل) يمكنه فقط:
  - قراءة بياناته الخاصة.
  - تعديل بياناته الخاصة.
  - حذف بياناته الخاصة.
  - إضافة بيانات مرتبطة بمعرفه الفريد فقط.

## توجيهات التطوير المستقبلي
يجب على أي تطوير قادم احترام هذا المنطق البرمجي وعدم كسر آلية التفريق بين الحفظ المحلي والسحابي، مع الحفاظ على سلامة سياسات الأمان المتبعة.
